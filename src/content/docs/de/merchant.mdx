### **Erste Schritte**

Stacks Pay ermöglicht es Händlern, Zahlungen in Kryptowährung schnell und sicher auf der Stacks-Blockchain zu akzeptieren. Egal, ob Sie Produkte verkaufen, Spenden annehmen oder Kunden Rechnungen stellen, Stacks Pay generiert codierte Zahlungslinks oder QR-Codes, die Ihre Kunden verwenden können, um Zahlungen an Ihre Stacks-Wallet zu senden.

Zahlungslinks können über unser SDK oder API generiert werden (Standorte werden später hinzugefügt). Während **Boom Wallet** das Generieren von Stacks Pay-Links unterstützt, kann jede Stacks-kompatible Wallet (wie **Xverse** oder **Leather**) verwendet werden, um Zahlungen zu empfangen. Außerdem kann jeder Händler, der Stacks Pay akzeptiert, es in seinen Zahlungsablauf integrieren.

### **Schritt 1: Richten Sie Ihre Stacks Wallet ein**

Bevor Sie Zahlungen akzeptieren können, benötigen Sie eine Stacks-Wallet, um Gelder zu empfangen. Wenn Sie noch keine haben, befolgen Sie diese Schritte:

1. **Wählen Sie eine Wallet**: Sie können jede Stacks-kompatible Wallet verwenden, um Zahlungen zu empfangen, wie [Xverse](https://www.xverse.app/), [Leather](https://leather.io/) oder [Boom Wallet](https://www.boomwallet.app).

2. **Erstellen Sie Ihre Wallet**: Laden Sie die Wallet Ihrer Wahl herunter und installieren Sie sie. Befolgen Sie dann die Anweisungen, um eine neue Wallet zu erstellen. Achten Sie darauf, Ihre Wiederherstellungsphrase sicher zu speichern, da sie für den Zugriff auf Ihre Gelder unerlässlich ist.

3. **Holen Sie sich Ihre Stacks-Adresse**: Nachdem Sie Ihre Wallet eingerichtet haben, kopieren Sie Ihre Stacks-Adresse. Es handelt sich in der Regel um eine lange Zeichenfolge, die mit "SP" beginnt (z. B. `SP2YDB...`). Dies ist die Adresse, die Sie verwenden werden, um Zahlungen zu erhalten.

### **Schritt 2: Erstellen Sie eine Zahlungsanforderung**

Um Zahlungen zu akzeptieren, müssen Sie einen Stacks Pay-Zahlungslink generieren. Diese codierten Zahlungslinks können mit unserem SDK oder API generiert werden (Standorte werden später hinzugefügt). Während **Boom Wallet** das Generieren von Stacks Pay-Links unterstützt, können Händler Stacks Pay in ihre eigenen Systeme integrieren.

So erstellen Sie einen Stacks Pay-Zahlungslink:

1. **Verwenden Sie das SDK oder die API**:
   - Verwenden Sie unser SDK oder die API, um einen Zahlungslink zu generieren, indem Sie die erforderlichen Zahlungsdetails angeben.
   - Geben Sie folgende Informationen ein:
     - **Empfänger**: BNS-Name oder Stacks-Adresse, an die die Zahlung gesendet wird.
     - **Betrag**: Der feste Betrag (in STX oder Token), wenn Sie ein Produkt verkaufen. Für Spenden oder offene Zahlungen können Sie dieses Feld leer lassen, damit der Kunde den gewünschten Betrag eingeben kann.
     - **Beschreibung**: Eine kurze Beschreibung der Zahlung (z. B. „Zahlung für Beratungsdienstleistungen“ oder „Spende zur Unterstützung unserer gemeinnützigen Organisation“).
     - **Stacks Pay ID (spId)**: Standardmäßig wird eine `spId` automatisch generiert. Sie können jedoch eine benutzerdefinierte `spId` angeben, um die Zahlung mit Ihren internen Bestellnummern oder Arbeitsabläufen zu verknüpfen.

   **Hinweis**: Dokumentation und Ressourcen für das SDK und die API werden in Kürze verfügbar sein. Bitte schauen Sie später erneut vorbei.

2. **Generieren Sie den codierten Zahlungslink**:
   - Das SDK oder die API erstellt basierend auf den bereitgestellten Informationen einen Bech32m-codierten Zahlungslink.
   - **Hinweis**: Stacks Pay-URLs sind immer codiert, um Kompatibilität, Sicherheit und Fehlererkennung zu gewährleisten. Der codierte Link sieht ungefähr so aus:
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Dieser codierte Link stellt sicher, dass die Zahlungsdetails sicher an die Wallet des Kunden übermittelt werden.

3. **Generieren Sie den QR-Code (optional)**:
   - Wenn Sie Transaktionen vor Ort abwickeln oder es mobilen Nutzern erleichtern möchten, können Sie auch einen QR-Code aus dem Zahlungslink generieren. Viele Tools und Bibliotheken bieten die Möglichkeit, einen QR-Code aus einer URL zu generieren.

### **Schritt 3: Teilen Sie die Zahlungsanforderung mit Ihrem Kunden**

Nachdem Sie den Stacks Pay-Link oder QR-Code generiert haben, können Sie ihn problemlos mit Ihren Kunden teilen.

1. **Per E-Mail oder Messaging**:
   - Kopieren Sie den codierten Zahlungslink und fügen Sie ihn in eine E-Mail oder Nachricht an Ihren Kunden ein.
   - Beispiel: „Bitte klicken Sie auf den unten stehenden Link, um Ihre Zahlung abzuschließen: [codierter Zahlungslink]“

2. **Integrieren Sie es in Ihre Website**:
   - Wenn Sie einen Online-Shop betreiben, können Sie den Zahlungslink als Schaltfläche auf Ihrer Website einbetten.
   - Beispiel: Fügen Sie eine „Mit Stacks bezahlen“-Schaltfläche hinzu, die den codierten Zahlungs-URL für ein nahtloses Kundenerlebnis verlinkt.

3. **Verwenden Sie einen QR-Code für persönliche Zahlungen**:
   - Für mobile oder persönliche Zahlungen zeigen Sie den generierten QR-Code an der Kasse an. Ihr Kunde kann den QR-Code mit seiner Wallet-App (z. B. Boom, Xverse oder Leather) scannen, um die Zahlung zu starten.

### **Schritt 4: Zahlungen in Ihrer Wallet empfangen**

Sobald Ihr Kunde die Zahlung abgeschlossen hat, werden die Gelder direkt an Ihre Stacks-Wallet gesendet. So können Sie die Zahlung nachverfolgen und überprüfen:

1. **Überprüfen Sie Ihre Wallet**:
   - Melden Sie sich bei Ihrer Wallet (Boom, Xverse, Leather usw.) an und überwachen Sie eingehende Transaktionen. Zahlungen sollten kurz nach der Bestätigung auf der Blockchain erscheinen.

2. **Überprüfen Sie die Zahlung**:
   - Überprüfen Sie die Transaktionsdetails, um sicherzustellen, dass der korrekte Betrag gesendet wurde. Die `spId` (Stacks Pay ID) kann Ihnen helfen zu bestätigen, dass die Zahlung Ihrer spezifischen Anfrage entspricht.

3. **Verfolgen Sie die Zahlung auf der Blockchain**:
   - Da Stacks Pay auf der Stacks-Blockchain arbeitet, können Sie jede Transaktion öffentlich in einem Blockchain-Explorer wie [Stacks Explorer](https://explorer.stacks.co) einsehen. Geben Sie einfach Ihre Stacks-Adresse ein, um den Zahlungsstatus zu sehen.

### **Schritt 5: Best Practices für die Nutzung von Stacks Pay**

Um einen reibungslosen Zahlungsprozess für Sie und Ihre Kunden zu gewährleisten, befolgen Sie diese Best Practices:

- **Verwenden Sie das SDK oder die API zur Zahlungserstellung**: Generieren Sie Stacks Pay-Zahlungslinks mit unserem SDK oder API, um eine nahtlose Integration in Ihre Systeme zu ermöglichen.

- **Verwenden Sie die `spId` zur Bestellverfolgung**: Verwenden Sie die `spId`, um Zahlungen mit Ihren internen Bestellnummern oder Arbeitsabläufen zu verknüpfen. Standardmäßig wird eine `spId` automatisch generiert, Sie können jedoch beim Erstellen des Zahlungslinks eine benutzerdefinierte `spId` angeben.

- **Überprüfen Sie die Zahlungsdetails**: Überprüfen Sie immer, ob die Empfängeradresse, der Betrag und die Beschreibung korrekt sind, bevor Sie den Zahlungslink teilen.

- **Teilen Sie Zahlungslinks sicher**: Wenn Sie Links per E-Mail, Messaging-Apps oder Websites teilen, stellen Sie sicher, dass sie sicher übertragen werden. Überprüfen Sie den Link immer, bevor Sie ihn senden, um Fehler zu vermeiden.

- **Überwachen Sie Ihre Zahlungen**: Behalten Sie Ihre Wallet im Auge, um eingehende Transaktionen zu verfolgen. Für Händler, die mehrere Zahlungen abwickeln, verwenden Sie die `spId`, um jede Transaktion effizient zu verfolgen und zu überprüfen.

- **Sichern Sie Ihre Wallet**: Für größere Beträge sollten Sie die Verwendung einer Hardware-Wallet oder die Aktivierung der Zwei-Faktor-Authentifizierung (2FA) in Betracht ziehen, um Ihre Gelder zu schützen.

- **Reagieren Sie schnell auf Zahlungsprobleme**: Wenn ein Kunde versehentlich den falschen Betrag sendet oder ein Zahlungsproblem auftritt, überprüfen Sie die Transaktion auf der Blockchain und gehen Sie eventuelle Bedenken umgehend an.
