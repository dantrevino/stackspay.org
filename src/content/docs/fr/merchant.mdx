### **Introduction**

StacksPay permet aux commerçants d'accepter des paiements en crypto-monnaie rapidement et en toute sécurité sur la blockchain Stacks. Que vous vendiez des produits, acceptiez des dons ou facturiez des clients, StacksPay génère des liens de paiement codés ou des codes QR que vos clients peuvent utiliser pour effectuer des paiements vers votre portefeuille Stacks.

Les liens de paiement peuvent être générés via notre SDK ou API (les emplacements seront ajoutés ultérieurement). Bien que **Boom Wallet** prenne en charge la génération de liens StacksPay, tout portefeuille compatible avec Stacks (comme **Xverse** ou **Leather**) peut être utilisé pour recevoir des paiements. De plus, tout commerçant qui accepte StacksPay peut l'intégrer dans son flux de paiement.

### **Étape 1 : Configurez votre portefeuille Stacks**

Avant de commencer à accepter des paiements, vous aurez besoin d'un portefeuille Stacks pour recevoir des fonds. Si vous n'en avez pas déjà un, suivez ces étapes :

1. **Choisissez un portefeuille** : Vous pouvez utiliser n'importe quel portefeuille compatible avec Stacks pour recevoir des paiements, comme [Xverse](https://www.xverse.app/), [Leather](https://leather.io/) ou [Boom Wallet](https://www.boomwallet.app).

2. **Créez votre portefeuille** : Téléchargez et installez le portefeuille de votre choix, puis suivez les instructions pour créer un nouveau portefeuille. Assurez-vous de stocker en toute sécurité votre phrase de récupération, car elle est essentielle pour accéder à vos fonds.

3. **Obtenez votre adresse Stacks** : Après avoir configuré votre portefeuille, copiez votre adresse Stacks. Il s'agit généralement d'une longue chaîne commençant par "SP" (par exemple, `SP2YDB...`). C'est l'adresse que vous utiliserez pour recevoir des paiements.

### **Étape 2 : Créez une demande de paiement**

Pour accepter des paiements, vous devrez générer un lien de paiement StacksPay. Vous pouvez générer ces liens de paiement codés en utilisant notre SDK ou API (les emplacements seront ajoutés ultérieurement). Bien que **Boom Wallet** prenne en charge la génération de liens StacksPay, les commerçants peuvent intégrer StacksPay dans leurs propres systèmes.

Voici comment générer un lien de paiement StacksPay :

1. **Utilisez le SDK ou l'API**:
   - Utilisez notre SDK ou API pour générer un lien de paiement en fournissant les détails nécessaires.
   - Entrez les informations suivantes :
     - **Destinataire** : nom BNS ou adresse Stacks où le paiement sera envoyé.
     - **Montant** : le montant fixe (en STX ou tokens) si vous vendez un produit. Pour les dons ou les paiements ouverts, vous pouvez laisser ce champ vide, permettant au client d'entrer le montant souhaité.
     - **Description** : une brève description du paiement (par exemple, « Paiement pour services de conseil » ou « Don pour soutenir notre association »).
     - **ID StacksPay (spId)** : par défaut, un `spId` sera généré automatiquement. Cependant, vous pouvez fournir un `spId` personnalisé pour lier le paiement à vos numéros de commande internes ou flux de travail.

   **Note** : La documentation et les ressources pour le SDK et l'API seront bientôt disponibles. Veuillez vérifier ultérieurement pour les mises à jour.

2. **Générez le lien de paiement codé**:
   - Le SDK ou l'API créera un lien de paiement codé en Bech32m basé sur les informations fournies.
   - **Note** : Les URL StacksPay sont toujours codées pour assurer la compatibilité, la sécurité et la détection des erreurs. Le lien codé ressemblera à ceci :
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Ce lien codé garantit que les détails du paiement sont transmis en toute sécurité au portefeuille du client.

3. **Générez le code QR (Optionnel)**:
   - Si vous gérez des transactions en personne ou souhaitez faciliter l'utilisation pour les mobiles, vous pouvez également générer un code QR à partir du lien de paiement. De nombreux outils et bibliothèques offrent la possibilité de générer un code QR à partir d'une URL.

### **Étape 3 : Partagez la demande de paiement avec votre client**

Une fois que vous avez généré le lien StacksPay ou le code QR, vous pouvez le partager facilement avec vos clients.

1. **Par e-mail ou messagerie**:
   - Copiez le lien de paiement codé et incluez-le dans un e-mail ou un message à votre client.
   - Exemple : « Veuillez cliquer sur le lien ci-dessous pour effectuer votre paiement : [Lien de paiement codé] »

2. **Intégrez-le sur votre site web**:
   - Si vous gérez une boutique en ligne, vous pouvez intégrer le lien de paiement comme un bouton sur votre site web.
   - Exemple : Ajoutez un bouton « Payer avec Stacks » qui renvoie à l'URL de paiement codée pour une expérience client transparente.

3. **Utilisez un code QR pour les paiements en personne**:
   - Pour les paiements mobiles ou en personne, affichez le code QR généré au point de vente. Votre client peut scanner le code QR avec son application de portefeuille (comme Boom, Xverse ou Leather) pour initier le paiement.

### **Étape 4 : Recevez les paiements dans votre portefeuille**

Une fois que votre client a effectué le paiement, les fonds seront envoyés directement à votre portefeuille Stacks. Voici comment suivre et vérifier le paiement :

1. **Vérifiez votre portefeuille**:
   - Connectez-vous à votre portefeuille (Boom, Xverse, Leather, etc.) et surveillez les transactions entrantes. Les paiements devraient apparaître peu de temps après avoir été confirmés sur la blockchain.

2. **Vérifiez le paiement**:
   - Consultez les détails de la transaction pour vous assurer que le montant correct a été envoyé. Le `spId` (ID StacksPay) peut vous aider à confirmer que le paiement correspond à votre demande spécifique.

3. **Suivez le paiement sur la blockchain**:
   - Étant donné que StacksPay fonctionne sur la blockchain Stacks, vous pouvez voir chaque transaction publiquement sur un explorateur de blockchain comme [Stacks Explorer](https://explorer.stacks.co). Il vous suffit d'entrer votre adresse Stacks pour voir le statut du paiement.

### **Étape 5 : Meilleures pratiques pour utiliser StacksPay**

Pour assurer un traitement des paiements fluide pour vous et vos clients, suivez ces meilleures pratiques :

- **Utilisez le SDK ou l'API pour générer les paiements** : Générez des liens de paiement StacksPay en utilisant notre SDK ou API pour une intégration transparente avec vos systèmes.

- **Utilisez le `spId` pour le suivi des commandes** : Utilisez le `spId` pour lier les paiements à vos numéros de commande internes ou flux de travail. Par défaut, un `spId` est généré automatiquement, mais vous pouvez fournir un `spId` personnalisé lors de la génération du lien de paiement.

- **Vérifiez les détails du paiement** : Vérifiez toujours que l'adresse du destinataire, le montant et la description sont corrects avant de partager le lien de paiement.

- **Partagez les liens de paiement en toute sécurité** : Lorsque vous partagez des liens par e-mail, applications de messagerie ou sites web, assurez-vous qu'ils sont transmis en toute sécurité. Prévisualisez toujours le lien avant de l'envoyer pour éviter les erreurs.

- **Surveillez vos paiements** : Gardez un œil sur votre portefeuille pour les transactions entrantes. Pour les commerçants traitant de multiples paiements, utilisez le `spId` pour suivre et vérifier chaque transaction efficacement.

- **Sécurisez votre portefeuille** : Pour des montants plus importants, envisagez d'utiliser un portefeuille matériel ou d'activer l'authentification à deux facteurs (2FA) pour protéger vos fonds.

- **Répondez rapidement aux problèmes de paiement** : Si un client envoie accidentellement le mauvais montant ou rencontre un problème de paiement, vérifiez la transaction sur la blockchain et réglez toute préoccupation en temps opportun.
