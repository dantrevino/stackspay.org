### **Começando**

O StacksPay permite que os comerciantes aceitem pagamentos em criptomoeda de forma rápida e segura na blockchain da Stacks. Esteja você vendendo produtos, aceitando doações ou faturando clientes, o StacksPay gera links de pagamento codificados ou códigos QR que seus clientes podem usar para fazer pagamentos em sua carteira Stacks.

Os links de pagamento podem ser gerados por meio de nosso SDK ou API (os locais serão adicionados posteriormente). Embora a **Boom Wallet** suporte a geração de links do StacksPay, qualquer carteira compatível com Stacks (como **Xverse** ou **Leather**) pode ser usada para receber pagamentos. Além disso, qualquer comerciante que aceite o StacksPay pode integrá-lo ao seu fluxo de trabalho de pagamento.

### **Etapa 1: Configure sua carteira Stacks**

Antes de começar a aceitar pagamentos, você precisará de uma carteira Stacks para receber fundos. Se você ainda não tiver uma, siga estas etapas:

1. **Escolha uma carteira**: Você pode usar qualquer carteira compatível com Stacks para receber pagamentos, como a [Boom Wallet](https://boom.money).

2. **Crie sua carteira**: Baixe e instale a carteira de sua escolha e siga as instruções para criar uma nova carteira. Certifique-se de armazenar com segurança sua frase de recuperação, pois ela é essencial para acessar seus fundos.

3. **Obtenha seu endereço Stacks**: Após configurar sua carteira, copie seu endereço Stacks. Geralmente, é uma longa sequência de caracteres que começa com "SP" (por exemplo, `SP2YDB...`). Este é o endereço que você usará para receber pagamentos.

### **Etapa 2: Crie uma solicitação de pagamento**

Para aceitar pagamentos, você precisará gerar um link de pagamento do StacksPay. Você pode gerar esses links de pagamento codificados usando nosso SDK ou API (os locais serão adicionados posteriormente). Embora a **Boom Wallet** suporte a geração de links do StacksPay, os comerciantes podem integrar o StacksPay em seus próprios sistemas.

Veja como gerar um link de pagamento do StacksPay:

1. **Use o SDK ou a API**:
   - Utilize nosso SDK ou API para gerar um link de pagamento, fornecendo os detalhes de pagamento necessários.
   - Insira as seguintes informações:
     - **Destinatário**: nome BNS ou endereço Stacks para onde o pagamento será enviado.
     - **Valor**: O valor fixo (em STX ou tokens) se você estiver vendendo um produto. Para doações ou pagamentos em aberto, você pode deixar este campo em branco, permitindo que o cliente insira o valor desejado.
     - **Descrição**: Uma breve descrição do pagamento (por exemplo, “Pagamento por serviços de consultoria” ou “Doação para apoiar nossa organização sem fins lucrativos”).
     - **ID do StacksPay (spId)**: Por padrão, um `spId` será gerado automaticamente. No entanto, você pode fornecer um `spId` personalizado para vincular o pagamento aos seus números de pedido ou fluxos de trabalho internos.

   **Observação**: A documentação e os recursos para o SDK e a API estarão disponíveis em breve. Volte para verificar as atualizações.

2. **Gere o link de pagamento codificado**:
   - O SDK ou a API criarão um link de pagamento codificado em Bech32m com base nas informações fornecidas.
   - **Observação**: as URLs do StacksPay são sempre codificadas para garantir compatibilidade, segurança e detecção de erros. O link codificado será algo como:
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Este link codificado garante que os detalhes do pagamento sejam transmitidos com segurança para a carteira do cliente.

3. **Gere o código QR (opcional)**:
   - Se você estiver lidando com transações presenciais ou quiser facilitar para os usuários de dispositivos móveis, também pode gerar um código QR a partir do link de pagamento. Muitas ferramentas e bibliotecas oferecem a opção de gerar um código QR a partir de uma URL.

### **Etapa 3: Compartilhe a solicitação de pagamento com seu cliente**

Depois de gerar o link ou o código QR do StacksPay, você pode compartilhá-lo facilmente com seus clientes.

1. **Por e-mail ou mensagem**:
   - Copie o link de pagamento codificado e inclua-o em um e-mail ou mensagem para seu cliente.
   - Exemplo: “Clique no link abaixo para concluir seu pagamento: [Link de pagamento codificado]”

2. **Incorpore em seu site**:
   - Se você administra uma loja online, pode incorporar o link de pagamento como um botão em seu site.
   - Exemplo: Adicione um botão “Pagar com Stacks” que vincule à URL de pagamento codificada para uma experiência do cliente perfeita.

3. **Use um código QR para pagamentos presenciais**:
   - Para pagamentos móveis ou presenciais, exiba o código QR gerado no checkout. Seu cliente pode escanear o código QR com o aplicativo de carteira (como Boom, Xverse ou Leather) para iniciar o pagamento.

### **Etapa 4: Receba pagamentos em sua carteira**

Depois que seu cliente concluir o pagamento, os fundos serão enviados diretamente para sua carteira Stacks. Veja como rastrear e verificar o pagamento:

1. **Verifique sua carteira**:
   - Faça login em sua carteira (Boom, Xverse, Leather, etc.) e monitore as transações recebidas. Os pagamentos devem aparecer logo após serem confirmados na blockchain.

2. **Verifique o pagamento**:
   - Verifique os detalhes da transação para garantir que o valor correto foi enviado. O `spId` (ID do StacksPay) pode ajudá-lo a confirmar que o pagamento corresponde à sua solicitação específica.

3. **Rastreie o pagamento na blockchain**:
   - Como o StacksPay opera na blockchain da Stacks, você pode visualizar todas as transações publicamente em um explorador de blockchain como o [Stacks Explorer](https://explorer.stacks.co). Basta inserir seu endereço Stacks para visualizar o status do pagamento.

### **Etapa 5: Práticas recomendadas para usar o StacksPay**

Para garantir um processamento de pagamento tranquilo para você e seus clientes, siga estas práticas recomendadas:

- **Use o SDK ou a API para geração de pagamentos**: Gere links de pagamento do StacksPay usando nosso SDK ou API para se integrar perfeitamente aos seus sistemas.

- **Utilize o `spId` para rastreamento de pedidos**: Use o `spId` para vincular pagamentos aos seus números de pedido ou fluxos de trabalho internos. Por padrão, um `spId` é gerado automaticamente, mas você pode fornecer um `spId` personalizado ao gerar o link de pagamento.

- **Verifique os detalhes do pagamento**: Sempre verifique novamente se o endereço do destinatário, o valor e a descrição estão corretos antes de compartilhar o link de pagamento.

- **Compartilhe links de pagamento com segurança**: Ao compartilhar links por e-mail, aplicativos de mensagens ou sites, verifique se eles são transmitidos com segurança. Sempre visualize o link antes de enviar para evitar erros.

- **Monitore seus pagamentos**: Fique de olho em sua carteira para transações recebidas. Para comerciantes que lidam com vários pagamentos, use o `spId` para rastrear e verificar cada transação com eficiência.

- **Proteja sua carteira**: Para valores maiores, considere usar uma carteira de hardware ou habilitar a autenticação de dois fatores (2FA) para proteger seus fundos.

- **Responda rapidamente a problemas de pagamento**: Se um cliente enviar acidentalmente o valor errado ou tiver um problema de pagamento, verifique a transação na blockchain e resolva quaisquer preocupações prontamente.

### **Exemplo: Aceitando pagamentos com o StacksPay**

Digamos que você administre uma loja online e queira aceitar pagamentos em criptomoeda. Veja como você pode usar o StacksPay para lidar com o processo:

1. **Crie a solicitação de pagamento**:
   - Use o SDK ou a API para gerar um link de pagamento.
   - Insira seu endereço Stacks como destinatário.
   - Especifique o valor do produto ou serviço.
   - Adicione uma descrição como “Pagamento do pedido nº 12345”.
   - Forneça um `spId` correspondente ao seu número de pedido interno (por exemplo, `Pedido12345`) para vincular o pagamento ao seu fluxo de trabalho de pedido.

2. **Compartilhe o link de pagamento**:
   - Integre o link de pagamento ao seu processo de checkout.
   - Os clientes podem clicar no link ou escanear um código QR para iniciar o pagamento.

3. **O cliente faz o pagamento**:
   - A carteira do cliente (Boom, Xverse, Leather, etc.) será aberta com os detalhes do pagamento preenchidos. Eles podem confirmar и concluir o pagamento.

4. **Verifique o pagamento**:
   - Você verifica sua carteira ou sistema para garantir que o pagamento foi recebido. O `spId` ajuda a confirmar que o pagamento corresponde ao pedido específico.

### **Próximas etapas para comerciantes**

Agora que você entende como criar e gerenciar solicitações de pagamento do StacksPay, é hora de integrá-lo ao seu negócio:

- **Use o SDK ou a API**: Comece a criar links de pagamento codificados para seus produtos, serviços ou doações usando nosso SDK ou API (os locais serão adicionados posteriormente).

- **Integre com seus sistemas**: Incorpore o StacksPay em seus fluxos de trabalho de pagamento existentes, utilizando o `spId` para rastrear pedidos e pagamentos.

- **Compartilhe links de pagamento com os clientes**: Seja por meio de seu site, e-mail ou códigos QR, facilite o pagamento dos clientes usando o StacksPay.

- **Monitore as transações**: Acompanhe todos os pagamentos recebidos e verifique-os usando sua carteira ou um explorador de blockchain.

Para obter mais informações sobre como os clientes usarão o StacksPay para fazer pagamentos, consulte nosso [Guia do usuário sobre como pagar usando o StacksPay](#como-pagar-usando-o-stackspay).
