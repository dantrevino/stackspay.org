### **Introdução**

O StacksPay permite que comerciantes aceitem pagamentos em criptomoedas de forma rápida e segura na blockchain Stacks. Seja vendendo produtos, aceitando doações ou faturando clientes, o StacksPay gera links de pagamento codificados ou códigos QR que seus clientes podem usar para fazer pagamentos para sua carteira Stacks.

Os links de pagamento podem ser gerados via nosso SDK ou API (locais serão adicionados posteriormente). Embora a **Boom Wallet** suporte a geração de links StacksPay, qualquer carteira compatível com Stacks (como **Xverse** ou **Leather**) pode ser usada para receber pagamentos. Além disso, qualquer comerciante que aceite o StacksPay pode integrá-lo em seu fluxo de pagamento.

### **Passo 1: Configure Sua Carteira Stacks**

Antes de começar a aceitar pagamentos, você precisará de uma carteira Stacks para receber fundos. Se ainda não tiver uma, siga estes passos:

1. **Escolha uma Carteira**: Você pode usar qualquer carteira compatível com Stacks para receber pagamentos, como [Xverse](https://www.xverse.app/), [Leather](https://leather.io/) ou [Boom Wallet](https://www.boomwallet.app).

2. **Crie Sua Carteira**: Baixe e instale a carteira de sua escolha e siga as instruções para criar uma nova carteira. Certifique-se de armazenar sua frase de recuperação com segurança, pois é essencial para acessar seus fundos.

3. **Obtenha Seu Endereço Stacks**: Após configurar sua carteira, copie seu endereço Stacks. Geralmente, é uma longa sequência começando com "SP" (por exemplo, `SP2YDB...`). Este é o endereço que você usará para receber pagamentos.

### **Passo 2: Crie uma Solicitação de Pagamento**

Para aceitar pagamentos, você precisará gerar um link de pagamento StacksPay. Você pode gerar esses links de pagamento codificados usando nosso SDK ou API (locais serão adicionados posteriormente). Embora a **Boom Wallet** suporte a geração de links StacksPay, comerciantes podem integrar o StacksPay em seus próprios sistemas.

Veja como gerar um link de pagamento StacksPay:

1. **Use o SDK ou API**:
   - Utilize nosso SDK ou API para gerar um link de pagamento fornecendo os detalhes necessários.
   - Insira as seguintes informações:
     - **Destinatário**: nome BNS ou endereço Stacks para onde o pagamento será enviado.
     - **Valor**: O valor fixo (em STX ou tokens) se você estiver vendendo um produto. Para doações ou pagamentos abertos, você pode deixar este campo em branco, permitindo que o cliente insira o valor desejado.
     - **Descrição**: Uma breve descrição do pagamento (por exemplo, “Pagamento por serviços de consultoria” ou “Doação para apoiar nossa organização sem fins lucrativos”).
     - **ID StacksPay (spId)**: Por padrão, um `spId` será gerado automaticamente. No entanto, você pode fornecer um `spId` personalizado para vincular o pagamento aos seus números de pedido internos ou fluxos de trabalho.

   **Nota**: A documentação e os recursos para o SDK e API estarão disponíveis em breve. Por favor, verifique novamente para atualizações.

2. **Gere o Link de Pagamento Codificado**:
   - O SDK ou API criará um link de pagamento codificado em Bech32m com base nas informações fornecidas.
   - **Nota**: As URLs do StacksPay são sempre codificadas para garantir compatibilidade, segurança e detecção de erros. O link codificado terá uma aparência semelhante a:
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Este link codificado garante que os detalhes do pagamento sejam transmitidos com segurança para a carteira do cliente.

3. **Gere o Código QR (Opcional)**:
   - Se você estiver lidando com transações presenciais ou quiser facilitar para usuários móveis, também pode gerar um código QR a partir do link de pagamento. Muitas ferramentas e bibliotecas oferecem a opção de gerar um código QR a partir de uma URL.

### **Passo 3: Compartilhe a Solicitação de Pagamento com Seu Cliente**

Uma vez que você tenha gerado o link do StacksPay ou código QR, pode compartilhá-lo facilmente com seus clientes.

1. **Via E-mail ou Mensagem**:
   - Copie o link de pagamento codificado e inclua-o em um e-mail ou mensagem para seu cliente.
   - Exemplo: “Por favor, clique no link abaixo para concluir seu pagamento: [Link de Pagamento Codificado]”

2. **Incorpore em Seu Site**:
   - Se você administra uma loja online, pode incorporar o link de pagamento como um botão em seu site.
   - Exemplo: Adicione um botão “Pagar com Stacks” que linka para a URL de pagamento codificada para uma experiência de cliente perfeita.

3. **Use um Código QR para Pagamentos Presenciais**:
   - Para pagamentos móveis ou presenciais, exiba o código QR gerado no ponto de venda. Seu cliente pode escanear o código QR com seu aplicativo de carteira (como Boom, Xverse ou Leather) para iniciar o pagamento.

### **Passo 4: Receba Pagamentos em Sua Carteira**

Depois que seu cliente concluir o pagamento, os fundos serão enviados diretamente para sua carteira Stacks. Veja como rastrear e verificar o pagamento:

1. **Verifique Sua Carteira**:
   - Faça login em sua carteira (Boom, Xverse, Leather, etc.) e monitore as transações recebidas. Os pagamentos devem aparecer pouco depois de serem confirmados na blockchain.

2. **Verifique o Pagamento**:
   - Verifique os detalhes da transação para garantir que o valor correto foi enviado. O `spId` (ID StacksPay) pode ajudá-lo a confirmar que o pagamento corresponde à sua solicitação específica.

3. **Acompanhe o Pagamento na Blockchain**:
   - Como o StacksPay opera na blockchain Stacks, você pode visualizar cada transação publicamente em um explorador de blockchain como o [Stacks Explorer](https://explorer.stacks.co). Basta inserir seu endereço Stacks para ver o status do pagamento.

### **Passo 5: Melhores Práticas para Usar o StacksPay**

Para garantir um processamento de pagamentos tranquilo para você e seus clientes, siga estas melhores práticas:

- **Use o SDK ou API para Gerar Pagamentos**: Gere links de pagamento StacksPay usando nosso SDK ou API para integrá-los perfeitamente aos seus sistemas.

- **Utilize o `spId` para Rastreamento de Pedidos**: Use o `spId` para vincular pagamentos aos seus números de pedido internos ou fluxos de trabalho. Por padrão, um `spId` é gerado automaticamente, mas você pode fornecer um `spId` personalizado ao gerar o link de pagamento.

- **Verifique os Detalhes do Pagamento**: Sempre verifique se o endereço do destinatário, valor e descrição estão corretos antes de compartilhar o link de pagamento.

- **Compartilhe Links de Pagamento com Segurança**: Ao compartilhar links via e-mail, aplicativos de mensagem ou sites, certifique-se de que sejam transmitidos com segurança. Sempre visualize o link antes de enviá-lo para evitar erros.

- **Monitore Seus Pagamentos**: Fique de olho em sua carteira para transações recebidas. Para comerciantes que lidam com múltiplos pagamentos, use o `spId` para rastrear e verificar cada transação de forma eficiente.

- **Proteja Sua Carteira**: Para valores maiores, considere usar uma carteira de hardware ou habilitar a autenticação de dois fatores (2FA) para proteger seus fundos.

- **Responda Rapidamente a Problemas de Pagamento**: Se um cliente enviar acidentalmente o valor errado ou enfrentar um problema de pagamento, verifique a transação na blockchain e aborde quaisquer preocupações em tempo hábil.
