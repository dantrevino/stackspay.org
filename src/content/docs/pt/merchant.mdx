### **Introdução**

O StacksPay permite que comerciantes aceitem pagamentos em criptomoedas de forma rápida e segura na blockchain Stacks. Quer esteja a vender produtos, aceitar doações ou faturar clientes, o StacksPay gera links de pagamento codificados ou códigos QR que os seus clientes podem usar para fazer pagamentos para a sua carteira Stacks.

Os links de pagamento podem ser gerados através do nosso SDK ou API (as localizações serão adicionadas posteriormente). Embora a **Boom Wallet** suporte a geração de links StacksPay, qualquer carteira compatível com Stacks (como **Xverse** ou **Leather**) pode ser usada para receber pagamentos. Além disso, qualquer comerciante que aceite o StacksPay pode integrá-lo no seu fluxo de pagamento.

### **Passo 1: Configure a Sua Carteira Stacks**

Antes de começar a aceitar pagamentos, precisará de uma carteira Stacks para receber fundos. Se ainda não tiver uma, siga estes passos:

1. **Escolha uma Carteira**: Pode usar qualquer carteira compatível com Stacks para receber pagamentos, como [Xverse](https://www.xverse.app/), [Leather](https://leather.io/) ou [Boom Wallet](https://www.boomwallet.app).

2. **Crie a Sua Carteira**: Descarregue e instale a carteira da sua escolha e siga as instruções para criar uma nova carteira. Certifique-se de guardar a sua frase de recuperação em segurança, pois é essencial para aceder aos seus fundos.

3. **Obtenha o Seu Endereço Stacks**: Após configurar a sua carteira, copie o seu endereço Stacks. Tipicamente, é uma longa sequência que começa com "SP" (por exemplo, `SP2YDB...`). Este é o endereço que usará para receber pagamentos.

### **Passo 2: Crie um Pedido de Pagamento**

Para aceitar pagamentos, precisará de gerar um link de pagamento StacksPay. Pode gerar estes links de pagamento codificados usando o nosso SDK ou API (as localizações serão adicionadas posteriormente). Embora a **Boom Wallet** suporte a geração de links StacksPay, os comerciantes podem integrar o StacksPay nos seus próprios sistemas.

Veja como gerar um link de pagamento StacksPay:

1. **Use o SDK ou API**:
   - Utilize o nosso SDK ou API para gerar um link de pagamento fornecendo os detalhes necessários.
   - Insira as seguintes informações:
     - **Destinatário**: nome BNS ou endereço Stacks para onde o pagamento será enviado.
     - **Montante**: O montante fixo (em STX ou tokens) se estiver a vender um produto. Para doações ou pagamentos abertos, pode deixar este campo em branco, permitindo que o cliente insira o montante desejado.
     - **Descrição**: Uma breve descrição do pagamento (por exemplo, “Pagamento por serviços de consultoria” ou “Doação para apoiar a nossa organização sem fins lucrativos”).
     - **ID StacksPay (spId)**: Por defeito, um `spId` será gerado automaticamente. No entanto, pode fornecer um `spId` personalizado para associar o pagamento aos seus números de encomenda internos ou fluxos de trabalho.

   **Nota**: A documentação e recursos para o SDK e API estarão disponíveis em breve. Por favor, verifique novamente para atualizações.

2. **Gere o Link de Pagamento Codificado**:
   - O SDK ou API criará um link de pagamento codificado em Bech32m com base nas informações fornecidas.
   - **Nota**: Os URLs do StacksPay são sempre codificados para garantir compatibilidade, segurança e deteção de erros. O link codificado terá um aspeto semelhante a:
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Este link codificado garante que os detalhes do pagamento são transmitidos com segurança para a carteira do cliente.

3. **Gere o Código QR (Opcional)**:
   - Se estiver a lidar com transações presenciais ou quiser facilitar para utilizadores móveis, também pode gerar um código QR a partir do link de pagamento. Muitas ferramentas e bibliotecas oferecem a opção de gerar um código QR a partir de um URL.

### **Passo 3: Partilhe o Pedido de Pagamento com o Seu Cliente**

Depois de gerar o link StacksPay ou código QR, pode facilmente partilhá-lo com os seus clientes.

1. **Via E-mail ou Mensagem**:
   - Copie o link de pagamento codificado e inclua-o num e-mail ou mensagem para o seu cliente.
   - Exemplo: “Por favor, clique no link abaixo para concluir o seu pagamento: [Link de Pagamento Codificado]”

2. **Incorpore no Seu Site**:
   - Se gerir uma loja online, pode incorporar o link de pagamento como um botão no seu site.
   - Exemplo: Adicione um botão “Pagar com Stacks” que liga ao URL de pagamento codificado para uma experiência de cliente perfeita.

3. **Use um Código QR para Pagamentos Presenciais**:
   - Para pagamentos móveis ou presenciais, exiba o código QR gerado no ponto de venda. O seu cliente pode digitalizar o código QR com a sua aplicação de carteira (como Boom, Xverse ou Leather) para iniciar o pagamento.

### **Passo 4: Receba Pagamentos na Sua Carteira**

Uma vez que o seu cliente conclua o pagamento, os fundos serão enviados diretamente para a sua carteira Stacks. Veja como rastrear e verificar o pagamento:

1. **Verifique a Sua Carteira**:
   - Inicie sessão na sua carteira (Boom, Xverse, Leather, etc.) e monitorize as transações recebidas. Os pagamentos devem aparecer pouco depois de serem confirmados na blockchain.

2. **Verifique o Pagamento**:
   - Verifique os detalhes da transação para garantir que o montante correto foi enviado. O `spId` (ID StacksPay) pode ajudá-lo a confirmar que o pagamento corresponde ao seu pedido específico.

3. **Acompanhe o Pagamento na Blockchain**:
   - Como o StacksPay opera na blockchain Stacks, pode visualizar cada transação publicamente num explorador de blockchain como o [Stacks Explorer](https://explorer.stacks.co). Basta inserir o seu endereço Stacks para ver o estado do pagamento.

### **Passo 5: Melhores Práticas para Utilizar o StacksPay**

Para garantir um processamento de pagamentos fluido para si e para os seus clientes, siga estas melhores práticas:

- **Utilize o SDK ou API para Gerar Pagamentos**: Gere links de pagamento StacksPay usando o nosso SDK ou API para integrar perfeitamente com os seus sistemas.

- **Utilize o `spId` para Rastreio de Encomendas**: Use o `spId` para associar pagamentos aos seus números de encomenda internos ou fluxos de trabalho. Por defeito, um `spId` é gerado automaticamente, mas pode fornecer um `spId` personalizado ao gerar o link de pagamento.

- **Verifique os Detalhes do Pagamento**: Verifique sempre se o endereço do destinatário, montante e descrição estão corretos antes de partilhar o link de pagamento.

- **Partilhe Links de Pagamento com Segurança**: Ao partilhar links via e-mail, aplicações de mensagens ou sites, certifique-se de que são transmitidos com segurança. Visualize sempre o link antes de o enviar para evitar erros.

- **Monitorize os Seus Pagamentos**: Fique atento à sua carteira para transações recebidas. Para comerciantes que lidam com múltiplos pagamentos, use o `spId` para rastrear e verificar cada transação de forma eficiente.

- **Proteja a Sua Carteira**: Para montantes maiores, considere utilizar uma carteira de hardware ou ativar a autenticação de dois fatores (2FA) para proteger os seus fundos.

- **Responda Rapidamente a Problemas de Pagamento**: Se um cliente enviar acidentalmente o montante errado ou enfrentar um problema de pagamento, verifique a transação na blockchain e aborde quaisquer preocupações atempadamente.
