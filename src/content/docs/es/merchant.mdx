### **Introducción**

StacksPay permite a los comerciantes aceptar pagos en criptomonedas de manera rápida y segura en la blockchain de Stacks. Ya sea que estés vendiendo productos, aceptando donaciones o facturando a clientes, StacksPay genera enlaces de pago codificados o códigos QR que tus clientes pueden usar para realizar pagos a tu billetera de Stacks.

Los enlaces de pago se pueden generar a través de nuestro SDK o API (las ubicaciones se agregarán más adelante). Aunque **Boom Wallet** admite la generación de enlaces de StacksPay, cualquier billetera compatible con Stacks (como **Xverse** o **Leather**) se puede usar para recibir pagos. Además, cualquier comerciante que acepte StacksPay puede integrarlo en su flujo de pagos.

### **Paso 1: Configura tu billetera de Stacks**

Antes de que puedas comenzar a aceptar pagos, necesitarás una billetera de Stacks para recibir fondos. Si aún no tienes una, sigue estos pasos:

1. **Elige una billetera**: Puedes usar cualquier billetera compatible con Stacks para recibir pagos, como [Xverse](https://www.xverse.app/), [Leather](https://leather.io/) o [Boom Wallet](https://www.boomwallet.app).

2. **Crea tu billetera**: Descarga e instala la billetera de tu elección, luego sigue las instrucciones para crear una nueva billetera. Asegúrate de almacenar de forma segura tu frase de recuperación, ya que es esencial para acceder a tus fondos.

3. **Obtén tu dirección de Stacks**: Después de configurar tu billetera, copia tu dirección de Stacks. Por lo general, es una cadena larga que comienza con "SP" (por ejemplo, `SP2YDB...`). Esta es la dirección que usarás para recibir pagos.

### **Paso 2: Crea una solicitud de pago**

Para aceptar pagos, necesitarás generar un enlace de pago de StacksPay. Puedes generar estos enlaces de pago codificados utilizando nuestro SDK o API (las ubicaciones se agregarán más adelante). Aunque **Boom Wallet** admite la generación de enlaces de StacksPay, los comerciantes pueden integrar StacksPay en sus propios sistemas.

Aquí te explicamos cómo generar un enlace de pago de StacksPay:

1. **Usa el SDK o API**:
   - Utiliza nuestro SDK o API para generar un enlace de pago proporcionando los detalles necesarios.
   - Ingresa la siguiente información:
     - **Destinatario**: nombre BNS o dirección de Stacks donde se enviará el pago.
     - **Cantidad**: el monto fijo (en STX o tokens) si estás vendiendo un producto. Para donaciones o pagos abiertos, puedes dejar este campo en blanco, permitiendo que el cliente ingrese la cantidad deseada.
     - **Descripción**: una breve descripción del pago (por ejemplo, “Pago por servicios de consultoría” o “Donación para apoyar nuestra organización sin fines de lucro”).
     - **ID de StacksPay (spId)**: Por defecto, se generará automáticamente un `spId`, pero puedes proporcionar un `spId` personalizado para vincular el pago a tus números de orden internos o flujos de trabajo.

   **Nota**: La documentación y los recursos para el SDK y API estarán disponibles pronto. Por favor, vuelve a consultar para actualizaciones.

2. **Genera el enlace de pago codificado**:
   - El SDK o API creará un enlace de pago codificado en Bech32m basado en la información proporcionada.
   - **Nota**: Las URL de StacksPay siempre están codificadas para garantizar compatibilidad, seguridad y detección de errores. El enlace codificado se verá algo así:
     ```
     stxpay1qpz9eyarqj8guvs49epj7wp6rszpe6sczxsrdsgx5uqzgkx5fw
     ```
     Este enlace codificado asegura que los detalles del pago se transmitan de forma segura a la billetera del cliente.

3. **Genera el código QR (Opcional)**:
   - Si manejas transacciones en persona o deseas facilitar a los usuarios móviles, también puedes generar un código QR a partir del enlace de pago. Muchas herramientas y bibliotecas ofrecen la opción de generar un código QR a partir de una URL.

### **Paso 3: Comparte la solicitud de pago con tu cliente**

Una vez que hayas generado el enlace de StacksPay o el código QR, puedes compartirlo fácilmente con tus clientes.

1. **Vía correo electrónico o mensajería**:
   - Copia el enlace de pago codificado e inclúyelo en un correo electrónico o mensaje a tu cliente.
   - Ejemplo: “Por favor, haz clic en el enlace a continuación para completar tu pago: [Enlace de Pago Codificado]”

2. **Incorpora en tu sitio web**:
   - Si administras una tienda en línea, puedes incorporar el enlace de pago como un botón en tu sitio web.
   - Ejemplo: Agrega un botón “Pagar con Stacks” que enlace a la URL de pago codificada para una experiencia de cliente sin inconvenientes.

3. **Usa un código QR para pagos en persona**:
   - Para pagos móviles o en persona, muestra el código QR generado en el punto de venta. Tu cliente puede escanear el código QR con su aplicación de billetera (como Boom, Xverse o Leather) para iniciar el pago.

### **Paso 4: Recibe pagos en tu billetera**

Una vez que tu cliente complete el pago, los fondos se enviarán directamente a tu billetera de Stacks. Aquí te mostramos cómo rastrear y verificar el pago:

1. **Verifica tu billetera**:
   - Inicia sesión en tu billetera (Boom, Xverse, Leather, etc.) y monitorea las transacciones entrantes. Los pagos deberían aparecer poco después de ser confirmados en la blockchain.

2. **Verifica el pago**:
   - Revisa los detalles de la transacción para asegurarte de que se haya enviado la cantidad correcta. El `spId` (ID de StacksPay) puede ayudarte a confirmar que el pago coincide con tu solicitud específica.

3. **Rastrea el pago en la blockchain**:
   - Dado que StacksPay opera en la blockchain de Stacks, puedes ver cada transacción públicamente en un explorador de blockchain como [Stacks Explorer](https://explorer.stacks.co). Simplemente ingresa tu dirección de Stacks para ver el estado del pago.

### **Paso 5: Mejores prácticas para usar StacksPay**

Para asegurar un procesamiento de pagos fluido tanto para ti como para tus clientes, sigue estas mejores prácticas:

- **Usa el SDK o API para generar pagos**: Genera enlaces de pago de StacksPay utilizando nuestro SDK o API para integrarlos sin problemas en tus sistemas.

- **Utiliza el `spId` para seguimiento de pedidos**: Usa el `spId` para vincular pagos a tus números de orden internos o flujos de trabajo. Por defecto, se genera automáticamente un `spId`, pero puedes proporcionar uno personalizado al generar el enlace de pago.

- **Verifica los detalles del pago**: Siempre revisa que la dirección del destinatario, la cantidad y la descripción sean correctas antes de compartir el enlace de pago.

- **Comparte enlaces de pago de forma segura**: Al compartir enlaces vía correo electrónico, aplicaciones de mensajería o sitios web, asegúrate de que se transmitan de forma segura. Siempre previsualiza el enlace antes de enviarlo para evitar errores.

- **Monitorea tus pagos**: Mantén un ojo en tu billetera para las transacciones entrantes. Para comerciantes que manejan múltiples pagos, utiliza el `spId` para rastrear y verificar cada transacción eficientemente.

- **Asegura tu billetera**: Para montos mayores, considera usar una billetera de hardware o habilitar la autenticación de dos factores (2FA) para proteger tus fondos.

- **Responde rápidamente a problemas de pago**: Si un cliente envía accidentalmente la cantidad incorrecta o experimenta un problema de pago, verifica la transacción en la blockchain y aborda cualquier inquietud de manera oportuna.
